# Getty Images — Submission Workflow

## Overview
LR Classic → Getty Metadata Studio → ExifTool (auto) → LR/Transporter → LR full-res export → DeepMeta → Getty ESP

---

## Step-by-Step Pipeline

### 1. Lightroom — Export small JPEGs for metadata generation
- **Preset name:** Getty Metadata Preview
- Format: JPEG
- Quality: 60
- Resize to Fit: Long Edge, 1500px
- Colour Space: sRGB
- No sharpening, no watermark
- Export to: `[T7]/Getty Submissions/[folder name]/_meta/`
- File size: ~300–500KB per image

### 2. Getty Metadata Studio (Claude.ai artifact)
- URL: Claude.ai conversation containing the app
- Fill in: Shoot Context, T7 Folder name
- Drop: small JPEGs from `_meta` folder
- Process in batches of 30–35 images
- Review metadata per image
- Click ⚡ Save & Embed → save CSV to `~/Desktop/Getty Drops/`
- Click ↓ Lightroom CSV → save for LR/Transporter import

### 3. ExifTool (automatic via watcher)
- Watcher monitors `~/Desktop/Getty Drops/` every 3 seconds
- Detects new `getty_metadata_*.csv` files
- Reads T7 folder path from CSV SourceFile column
- Runs: `exiftool -csv="[file]" -sep ";" -overwrite_original "[folder]"`
- Moves processed CSV to `~/Desktop/Getty Drops/Done/`
- Sends macOS notification: "Metadata embedded successfully"
- Log: `~/Library/Logs/getty_watcher.log`

### 4. LR/Transporter — Import metadata to catalog
- Open LR/Transporter
- Load the Lightroom CSV (separate from ExifTool CSV)
- Imports: Title, Caption, Keywords into LR catalog
- Keywords must be semicolon-separated (Transporter rejects comma-separated)
- All keywords must be pre-imported into LR keyword catalogue first
  (Metadata → Import Keywords)

### 5. Lightroom — Full-res export for Getty
- Select images with metadata now in catalog
- Export preset: Getty Full Res
- Format: JPEG, Quality: 100 (or per Getty spec)
- Metadata bakes in from catalog during export
- Export to: `[T7]/Getty Submissions/[folder name]/`

### 6. DeepMeta — Review and submit
- Add the full-res JPEG folder
- Verify: Title, Description, Keywords per image
- Set: Country of Shoot = United Kingdom, City = London
- Set: Number of People = No People (for architectural shots)
- Use lightbulb icon for keyword suggestions across batch
- Submit to Getty ESP

## File Naming Convention
- T7 path: `/Volumes/Samsung PSSD T7/Getty Submissions/`
- Folder format: `YYYY-MM-DD [Location Name]`
- Example: `2026-03-07 High Street Kensington`
- Meta subfolder: `2026-03-07 High Street Kensington Metadata`

## CSV Formats

### ExifTool CSV (generated by app)
```
SourceFile,IPTC:ObjectName,IPTC:Caption-Abstract,IPTC:Keywords
/Volumes/Samsung PSSD T7/Getty Submissions/[folder]/DSC_0001.jpg,Title Here,"Caption here",Keyword1;Keyword2;Keyword3
```
- SourceFile = full T7 path
- Keywords = semicolon-separated

### Lightroom CSV (generated by app)
```
Filename,Title,Caption,Keywords
DSC_0001.jpg,Title Here,"Caption here",Keyword1;Keyword2;Keyword3
```
- Filename only (no path)
- Keywords = semicolon-separated

# Restart watcher

| Recommended per drop | 30–35 images | ~5 min |
| Getty submission limit | 100 images | — |

## Known Issues & Fixes
- **ExifTool CORS error:** Fixed by running app via local Python server or Claude.ai artifact
- **LR/Transporter keyword drop:** Keywords must be pre-imported into LR catalogue
- **DeepMeta title truncation:** Keep titles under 60 characters
- **Watcher `declare -A` error:** Fixed — old bash doesn't support associative arrays, removed
- **Mac SSD full:** Run `sudo purge` then restart to clear purgeable space
